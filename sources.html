Source records

The site is designed to point families toward original records, not replace them.

Phase 1 source: Library of Congress map atlas

The first index uses the Library of Congress 1909 Cherokee Nation atlas by Indian Territory Map Co. LOC describes it as cadastral township maps showing names, statuses, allotment numbers, and other geographical features.

Dawes / NARA allotment records

For Five Tribes research, the strongest next source layer is Dawes/NARA records. Dawes land allotment jackets can include the applicant’s name, tribe, enrollment number, physical land location, legal description, improvements, plats, correspondence, and contested-selection notices.

Open NARA Dawes records guide

County clerk records

After a legal description is found, county land records may show what happened later: deeds, mortgages, releases, sheriff deeds, tax deeds, oil and gas leases, mineral records, liens, and other title-chain documents.

Oklahoma Historical Society land records note

BIA / LTRO trust or restricted land records

If land may be trust or restricted Indian land, county records are not enough. BIA Land Title and Records Offices maintain title conveyances and encumbrances for trust and restricted Indian land, and BIA’s Indian Land Record of Title can include patents, deeds, probate orders, leases, rights-of-way, cadastral surveys, plats, and other title documents requiring federal approval.

Dawes quick-search index layer

The new homepage Dawes search is a fast lead index, not a final proof system. It is built for the fields most useful to families at the start of research: name, tribe/Nation, enrollment category, roll/enrollment number, census card number, age, sex, blood degree, source, and notes.

The index file is data/dawes_index.json. Volunteers can prepare reviewed rows in data/dawes_index_template.csv, convert them with tools/build_dawes_index_from_csv.py, and redeploy the updated JSON. Keep source links and notes with every row so users know where to verify the match.

Fast data layers before full map indexing

The project does not have to wait for every township map to be transcribed. Families and reviewers can add source-linked clues now and match them to map entries later.

  • Dawes baseline: name, tribe/Nation, roll number, census card number, enrollment category, age, sex, and blood degree.
  • Allotment-packet status: packet ordered, packet received, packet not found, or still needed.
  • County legal-event clues: book/page, instrument number, probate case, sheriff sale, tax deed, mortgage, release, oil/gas lease, or court judgment.
  • BIA/LTRO clues: title status report, patent, deed, probate order, lease, right-of-way, plat, or file number.
  • Pattern tags: minor, guardian, tax sale, sheriff sale, mortgage, probate, restriction issue, missing approval, oil/gas, right-of-way, or unknown.

Official source connector hub

The homepage now includes an Official Source Lookup section. It combines live API-capable sources with official linkout/search paths so users can keep moving even when a source does not expose a simple public API.

  • NARA Catalog API: server-side connector at /api/nara-search. Requires a Cloudflare Pages secret named NARA_API_KEY. The API key must not be placed in browser JavaScript.
  • Library of Congress JSON/YAML API: connector at /api/loc-search for map/source leads.
  • OHS Dawes Rolls: official linkout/search path for first name, last name, tribal nation, roll number, and card number.
  • BIA/LTRO: official title-record request path for trust/restricted land records and Title Status Reports.
  • BLM GLO Records: official federal land-record linkout for patents, surveys, land-status records, tract books, and township land catalog research.
  • NARA AWS bulk dataset: planned offline/bulk metadata path for larger import work, instead of scraping the live NARA API.

v0.42 public source-lead connectors

This build adds no-key or low-friction source systems that make the site useful while map indexing continues.

  • Chronicling America / LOC: historic newspaper source leads for ancestor names, tax-sale notices, sheriff sales, guardian sales, probate notices, oil/gas notices, and allotment references.
  • Federal Register: public API source leads for BIA notices, tribal ordinances, land acquisitions, Indian Affairs rules, environmental notices, and official PDFs.
  • Census Geocoder / TIGERweb: address or coordinate leads for county/tract/block and Census tribal geography layers, including federal reservations, off-reservation trust lands, and Oklahoma Tribal Statistical Areas.
  • Record request packet builder: copy-paste request text for NARA, OHS, county clerks, BIA/LTRO, and BLM/GLO based on the user's clues.

Census geography results are source leads only. They do not prove ownership, title, trust/restricted status, or final Indian Country legal status.

Verification rule

All search results should be treated as research leads until verified with original source records. The goal is to make the trail easier to follow, not to replace official records.

Data-source roadmap

Possible official/public data feeds

  • NARA Catalog API: official live lookup for archival descriptions, authority records, digital-object metadata, OCR/extracted text, and source links. Use it for lookups and printed leads, not bulk scraping.
  • NARA AWS Catalog dataset: best bulk metadata path for later RG 75 / RG 48 import work.
  • Library of Congress JSON/YAML API: public API for map metadata, item/resource data, images, and source links.
  • Oklahoma Historical Society Dawes database: official state search reference for names, tribe, roll number, card number, and packet-order paths.
  • BIA/LTRO: official route for trust/restricted title documents, patents, deeds, probate orders, leases, rights-of-way, plats, and Title Status Reports; generally a record-request path, not a public bulk API.
  • BLM GLO Records: official federal land-record path for patents, survey plats, field notes, land-status records, tract books, and township catalog leads.

All automated imports should preserve source attribution and keep results labeled as research leads unless verified against original records.
