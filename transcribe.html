Help index the maps

The next goal is to turn public map images into verified searchable records without exposing private family information.

Why indexing is needed

The Phase 1 source is the 1909 Library of Congress Cherokee Nation atlas. The maps can show names, statuses, and allotment numbers, but those details are in map images. To search by family name or allotment number, the map text has to be transcribed, checked, and tied back to the original map page.

Safe transcription workflow

  1. Choose a map page. Start with township and range, such as T21N R12E.
  2. Open the original source. Review the map at the Library of Congress.
  3. Transcribe only what appears on the historical public record. Do not add private family stories or living-person information.
  4. Record uncertainty. Use question marks, notes, and confidence levels when handwriting or map text is unclear.
  5. Review before publishing. A row should not become a public searchable family result until it has source links and review status.

Human section-entry workflow

The OCR helper is now treated as optional. The preferred accuracy workflow is section-first and human-reviewed: create or open a section crop, read the names and allotment numbers from the image, then use the local Section Entry Helper to build approved JSON rows.

This approach keeps the fixed legal land unit — township, range, and section — as the anchor. Humans read the variable text inside each section before anything becomes public/searchable.

v0.19 human section workflow

Generate all 36 section crops from one LOC page without OCR, then use the local Section Entry Workbench to enter verified rows section by section.

python tools\map_indexing_agent.py --page 29 --mode sections --sections all --preprocess soft --section-padding 90 --crops-only

Open tools/section_entry.html, load the section manifest, select township/range/section, load the crop, and export approved rows.

Generate organized crops

For human transcription, use the v0.19 command to generate all 36 section crops organized by township/range/section: python tools\map_indexing_agent.py --page 29 --mode sections --sections all --preprocess soft --section-padding 240 --crops-only --output-layout trs.

v0.21 manual grid calibration

If section crops are shifted or missing part of the section, use tools/grid_calibrator.html. Load the full source map image, drag the outside grid lines first, distribute the internal lines, then export the calibration JSON.

python tools\map_indexing_agent.py --page 29 --mode sections --sections all --preprocess soft --crops-only --output-layout trs --manual-grid-json "data\grid_calibrations\p029_T24N_R14E_grid.json" --section-padding 80 --save-grid-debug

v0.22 one-folder township workbench

The easier workflow is now: calibrate once, generate one township/range workbook folder, then import that folder into tools/township_workbench.html. The workbench shows the source map, all 36 sections, duplicate checks, section status, and approved-row export from one page.

python tools\map_indexing_agent.py --page 29 --mode sections --sections all --preprocess soft --crops-only --output-layout workbook --manual-grid-json "data\grid_calibrations\p029_T24N_R14E_grid.json" --section-padding 80
